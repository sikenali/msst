# 修复说明：大乐透号码生成问题

## 问题一：运数/运式修改后有道按钮生成问题

### 问题描述
在大乐透标签下修改"运数"（注数）和"运式"（模式）后，点击"有道"按钮生成号码时，没有按照修改后的注数和模式生成对应的号码。

### 根本原因
`IconModal.vue` 组件中存在状态同步问题：

1. **组件初始化时过早获取引用**：在组件顶层调用 `useUserSelections()` 获取的引用可能指向双色球的状态，而不是大乐透的
2. **彩种类型同步时机**：虽然在 `watch` 中调用了 `setCurrentType(props.lotteryType)`，但之前获取的引用已经固定指向双色球
3. **保存时未确保类型正确**：点击"确定"按钮保存时，没有确保 `currentType` 已正确设置为大乐透

### 修复方案

#### 修改文件
`src/components/IconModal.vue`

#### 关键修改点

1. **移除顶层的 composable 解构**
   - 删除了在组件初始化时获取的 `useUserSelections()` 引用
   - 改为在需要时才动态调用

2. **在 watch 中正确获取引用**
   ```typescript
   watch(() => props.visible, (val) => {
     if (val) {
       // 先同步彩种类型
       if (props.lotteryType) {
         setCurrentType(props.lotteryType)
       }
       
       // 类型同步后再获取当前彩种的数据
       const { userNotes, userMode } = useUserSelections()
       
       // 恢复注数/模式
       if (props.type === 'yunshu') {
         notesCount.value = userNotes.value
       }
       if (props.type === 'yunshi') {
         yunshiMode.value = userMode.value
       }
     }
   })
   ```

3. **在保存时确保类型正确**
   ```typescript
   function handleNotesConfirm() {
     // 确保在正确的彩种类型下保存
     if (props.lotteryType) {
       setCurrentType(props.lotteryType)
     }
     const { setNotes, userNotes } = useUserSelections()
     setNotes(notesCount.value)
     handleClose()
   }
   
   function handleYunshiConfirm() {
     // 确保在正确的彩种类型下保存
     if (props.lotteryType) {
       setCurrentType(props.lotteryType)
     }
     const { setMode, userMode } = useUserSelections()
     setMode(yunshiMode.value)
     handleClose()
   }
   ```

## 问题二：选中号码后生成都是上一次的旧号码

### 问题描述
大乐透选中红球和蓝球后，每次生成都是上一次选中的号码。刷新页面和返回后，需要清理缓存，重新选中才生效。

### 根本原因

1. **`clearAll()` 不清除红球蓝球**：原有的 `clearAll()` 函数只清除生日、星座、生辰、幸运数，故意保留了红球蓝球
2. **用户期望与实现不符**：用户期望刷新/返回后重新选择号码，但旧号码仍保留在全局状态中
3. **切换彩种时未清除旧号码**：从双色球切换到大乐透时，双色球的号码没有被清除

### 修复方案

#### 修改文件
1. `src/views/ResultPage.vue`
2. `src/views/HomePage.vue`
3. `src/composables/useUserSelections.ts`（已有 `clearAllIncludingRedBlue` 函数，之前未被使用）

#### 关键修改点

**1. ResultPage.vue - 返回时清除所有数据**

```typescript
// 修改前
const { userNotes, setNotes, userMode, setMode, clearAll } = useUserSelections()

function handleBack() {
  // 返回主页时只清除生日、星座、生辰、幸运数，保留红球蓝球选中状态
  clearAll()
  router.push({ path: '/', query: { type: lotteryType.value, autoGenerate: '1' } })
}

// 修改后
const { userNotes, setNotes, userMode, setMode, clearAllIncludingRedBlue } = useUserSelections()

function handleBack() {
  // 返回主页时清除所有数据（包括号码），确保用户重新选择
  clearAllIncludingRedBlue()
  router.push({ path: '/', query: { type: lotteryType.value, autoGenerate: '1' } })
}
```

**2. ResultPage.vue - onMounted 时清除所有数据**

```typescript
// 修改前
// 刷新页面时只清除生日、星座、生辰、幸运数，保留红球蓝球选中状态
clearAll()

// 修改后
// 刷新页面时清除所有数据（包括号码），确保重新选择
clearAllIncludingRedBlue()
```

**3. HomePage.vue - 切换彩种时清除旧号码**

```typescript
// 修改前
watch(lotteryType, (newType) => {
  setCurrentType(newType)
  setCurrentLotteryType(newType)
}, { immediate: true })

// 修改后
const { ..., clearRedBlueNumbers } = useUserSelections()

watch(lotteryType, (newType) => {
  setCurrentType(newType)
  setCurrentLotteryType(newType)
  // 切换彩种时清除旧的号码，避免使用另一个彩种的号码
  clearRedBlueNumbers()
}, { immediate: true })
```

### 清除函数说明

`useUserSelections.ts` 中提供了三个清除函数：

| 函数名 | 清除范围 | 使用场景 |
|--------|----------|----------|
| `clearAll()` | 只清除生日、星座、生辰、幸运数 | 保留号码状态，只清除玄学数据 |
| `clearRedBlueNumbers()` | 只清除红球和蓝球 | 切换彩种时清除旧号码 |
| `clearAllIncludingRedBlue()` | 清除所有数据（包括号码、注数、模式） | 返回主页、刷新页面时完全重置 |

## 测试步骤

### 测试场景 1：修改运数（注数）
1. 切换到**大乐透**标签

7. ✅ **预期结果**：按照选择的模式生成对应的大乐透号码
   - 复式：前区>5个 或 后区>2个

### 测试场景 3：选中号码后生成

4. 点击**有道**按钮生成号码

6. ✅ **预期结果**：返回后号码被清除，需要重新选择

2. 切换到**大乐透**标签

## 技术细节

### 状态管理机制
- 使用全局响应式状态（`ref`）存储双色球和大乐透的独立状态
- 通过 `currentType` 变量动态切换当前激活的彩种
- `useUserSelections()` 根据 `currentType` 返回对应的响应式引用

### 关键文件
- `src/components/IconModal.vue` - 运数/运式弹框组件（修复问题一）
- `src/views/HomePage.vue` - 主页，包含生成按钮和彩种切换（修复问题二）
- `src/views/ResultPage.vue` - 结果页，包含返回逻辑（修复问题二）
- `src/composables/useUserSelections.ts` - 全局状态管理
- `src/composables/useLottery.ts` - 号码生成算法
