# 修复说明：大乐透运数/运式修改后有道按钮生成问题

## 问题描述
在大乐透标签下修改"运数"（注数）和"运式"（模式）后，点击"有道"按钮生成号码时，没有按照修改后的注数和模式生成对应的号码。

## 根本原因
`IconModal.vue` 组件中存在状态同步问题：

1. **组件初始化时过早获取引用**：在组件顶层调用 `useUserSelections()` 获取的引用可能指向双色球的状态，而不是大乐透的
2. **彩种类型同步时机**：虽然在 `watch` 中调用了 `setCurrentType(props.lotteryType)`，但之前获取的引用已经固定指向双色球
3. **保存时未确保类型正确**：点击"确定"按钮保存时，没有确保 `currentType` 已正确设置为大乐透

## 修复方案

### 修改文件
`src/components/IconModal.vue`

### 关键修改点

1. **移除顶层的 composable 解构**
   - 删除了在组件初始化时获取的 `useUserSelections()` 引用
   - 改为在需要时才动态调用

2. **在 watch 中正确获取引用**
   ```typescript
   watch(() => props.visible, (val) => {
     if (val) {
       // 先同步彩种类型
       if (props.lotteryType) {
         setCurrentType(props.lotteryType)
       }
       
       // 类型同步后再获取当前彩种的数据
       const { userNotes, userMode } = useUserSelections()
       
       // 恢复注数/模式
       if (props.type === 'yunshu') {
         notesCount.value = userNotes.value
       }
       if (props.type === 'yunshi') {
         yunshiMode.value = userMode.value
       }
     }
   })
   ```

3. **在保存时确保类型正确**
   ```typescript
   function handleNotesConfirm() {
     // 确保在正确的彩种类型下保存
     if (props.lotteryType) {
       setCurrentType(props.lotteryType)
     }
     const { setNotes, userNotes } = useUserSelections()
     setNotes(notesCount.value)
     handleClose()
   }
   
   function handleYunshiConfirm() {
     // 确保在正确的彩种类型下保存
     if (props.lotteryType) {
       setCurrentType(props.lotteryType)
     }
     const { setMode, userMode } = useUserSelections()
     setMode(yunshiMode.value)
     handleClose()
   }
   ```

4. **添加调试日志**
   - 在保存运数/运式时输出日志，方便调试
   - 显示保存的值和保存后的全局状态

## 测试步骤

### 测试场景 1：修改运数（注数）
1. 切换到**大乐透**标签
2. 点击左侧浮动面板的**乌龟图标** → 展开菜单
3. 点击**运数**（数字图标）
4. 修改注数为 **10**
5. 点击"确定"按钮
6. 点击**有道**按钮
7. ✅ **预期结果**：生成 10 注大乐透号码

### 测试场景 2：修改运式（模式）
1. 切换到**大乐透**标签
2. 点击左侧浮动面板的**乌龟图标** → 展开菜单
3. 点击**运式**（蜡烛图标）
4. 选择**复式**或**胆拖**模式
5. 点击"确定"按钮
6. 点击**有道**按钮
7. ✅ **预期结果**：按照选择的模式生成对应的大乐透号码
   - 复式：前区>5个 或 后区>2个
   - 胆拖：显示胆码和拖码

### 测试场景 3：同时修改运数和运式
1. 切换到**大乐透**标签
2. 点击**运数**，修改为 **8**，点击确定
3. 点击**运式**，选择**胆拖**，点击确定
4. 点击**有道**按钮
5. ✅ **预期结果**：生成 8 注大乐透胆拖号码

### 测试场景 4：在双色球和大乐透间切换
1. 在**双色球**下设置运数为 5，运式为单式
2. 切换到**大乐透**
3. 修改运数为 12，运式为复式
4. 点击**有道**按钮
5. ✅ **预期结果**：生成 12 注大乐透复式号码（不影响双色球的设置）

## 技术细节

### 状态管理机制
- 使用全局响应式状态（`ref`）存储双色球和大乐透的独立状态
- 通过 `currentType` 变量动态切换当前激活的彩种
- `useUserSelections()` 根据 `currentType` 返回对应的响应式引用

### 关键文件
- `src/components/IconModal.vue` - 运数/运式弹框组件（本次修复）
- `src/composables/useUserSelections.ts` - 全局状态管理
- `src/views/HomePage.vue` - 主页，包含生成按钮逻辑
- `src/composables/useLottery.ts` - 号码生成算法
